# 🚀 Reporte de Actualizaciones CitaLocal PWA
**Fecha:** 3 de Marzo de 2026

## 1. 📊 Mejoras en Dashboard y Estadísticas
- Se agregó navegación por fecha nativa permitiendo saltar a cualquier día usando un calendario visual interactivo.
- Las tarjetas de "Procedimientos" en las Estadísticas ahora son Acordeones (plegables). Al hacer clic en ellas, se despliega la lista detallada de pacientes de ese procedimiento 1 a 1.
- El detalle individual muestra el Nombre Real del Paciente, Número de Historia Clínica, Médico Tratante y Hora en orden cronológico.
- **Corrección de Lógica CEMA:** Anteriormente los doctores individuales veían las estadísticas globales de la clínica. Ahora la app consulta el correo, cruza con la base de datos `providers` y solo le muestra a cada doctor los números correspondientes a su cuenta particular. El único que ve números globales es el rol maestro "CEMA".
- **Limpieza de Citas Pendientes**: Solo se contabilizan en el número de procedimientos o estadísticas aquellas citas cuyo estatus sea `attended` (Atendidas).

## 2. 🗓 Agilitación de la Agenda y Visuales
- Se implementó la limpieza del molesto prefijo `"Dr(a)."` repetido en toda la aplicación (tanto el Dashboard individual como el Perfil). 
- Se removió el botón flotante "Salir" del Layout inferior general para prevenir accidentes de toques fantasmas, moviéndolo a una zona más segura.

## 3. 👤 Rediseño de la Vista de Perfil
- Refactorización completa de la pantalla "Mi Perfil".
- **Auto-Identificación Mejorada**: La aplicación busca el nombre real en la tabla de base de datos (`providers`); si no se encuentra allí o está en blanco, usa automáticamente el nombre y apellido con los que el usuario está registrado en Google.
- **Sincronización de Avatar**: La foto de perfil original subida por el usuario a su cuenta de Gmail ahora es extraída por Auth y expuesta con un marco curvo en la app.
- **Preferencias Visuales (Soporte Nativo Dark Mode / Light Mode)**:
  - Se creó un Componente Intermedio `ThemeProvider`.
  - Se acopló el esquema `@theme` nativo de **Tailwind CSS v4** mediante `custom-variant dark`.
  - Ahora cualquier médico desde su sección de Mi Perfil puede elegir un tema claro obligadamente, un tema oscuro para uso nocturno (para descansar la vista sin que brille la pantalla) o que la PWA siga el comportamiento inteligente del Teléfono.

## 4. 🪲 Reparación de Sincronizador Backend (N8N Webhook)
- **Problema Detectado**: Cuando en CitaLocal Original se eliminaban procedimientos viejos de una cita, Supabase (el backend de la PWA) no eliminaba los procedimientos anteriores sino que agregaba y acumulaba ambas especialidades bajo la misma cita (ej. *Cita 2157 | HC 1690* poseía erróneamente Láser y Terapia al mismo tiempo).
- **Solución Aplicada**: Se alteró el código primario del bot de sincronización (`sync_procedures_to_supabase.cjs`) que reacciona a los pulsos de n8n.
- A partir de hoy, en el ciclo de carga cada vez que se detectan relaciones actualizadas, la nube **ELIMINA FÍSICAMENTE** las asociaciones viejas de esas citas y escribe solo la confirmación actualizada final para mantener 100% la paridad entre tu sistema interno Desktop y la versión PWA móvil web. 
- Se corrió el script de migración masiva reparando todas las citas corruptas existentes sin generar duplicidades.

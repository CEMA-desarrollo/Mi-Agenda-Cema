# Actualización del Sistema PWA - 04/03/2026

En esta jornada se han corregido errores críticos relacionados con las Notificaciones Push, la sincronización de datos y la experiencia de usuario en dispositivos móviles.

## 1. Notificaciones Push y Permisos
- **Diagnóstico Preciso:** Se mejoraron los mensajes de error en el Perfil. Ahora el sistema informa exactamente si el error es por permisos del navegador, timeout del Service Worker o falta de vinculación del correo del usuario.
- **Corrección de UUID:** Se bloqueó la posibilidad de enviar un ID inválido ("unknown") a Supabase cuando el usuario no tiene un médico asociado, evitando errores de sintaxis en la base de datos.
- **Mapeo de Datos:** Se actualizaron los campos en el script de envío (`notify_push.cjs`) para coincidir con la estructura real de Supabase:
    - `patients.name` → `first_name` y `last_name`.
    - `appointments.date` → `start_time`.
- **Relación de Médicos:** Se implementó un "JOIN" para resolver el UUID de Supabase del médico basándose en su `local_id` de CitaLocal, permitiendo encontrar la suscripción del teléfono correctamente.

## 2. Automatización n8n
- **Flujo de Notificación:** Se vinculó el script de envío de notificaciones al final del proceso de sincronización local (`webhook-sync.cjs`).
- **Filtro de Seguridad:** El sistema ahora solo notifica citas que tengan `push_sent = false` y cuya fecha de inicio sea de **hoy en adelante**, evitando el envío de notificaciones por citas antiguas del pasado.

## 3. Mejoras en la Interfaz (UI/UX)
- **Control de Notificaciones:** Se reemplazó el botón de "Activar" por un **Interruptor (Toggle)** más intuitivo en el Perfil, que muestra el estado real (encendido/apagado) según la base de datos.
- **Calendario Móvil:** Se eliminó la visualización redundante de la hora dentro de los eventos del calendario para mejorar la legibilidad en pantallas pequeñas.

## 4. Infraestructura y Despliegue
- **Rutas en Vercel:** Se creó el archivo `vercel.json` para corregir los errores 404 que ocurrían al recargar páginas como "Estadísticas" o "Perfil" desde el navegador.
- **Iconos Android:** Se actualizó la configuración de Vite para asegurar que los iconos nuevos se reflejen correctamente en dispositivos Android tras la instalación.
- **Actualización Inmediata:** Se configuró el Service Worker para forzar la actualización de la App (`SKIP_WAITING`) en cuanto el usuario presiona el botón de recarga.

---
**Estado Actual:** El sistema de notificaciones está verificado y funcionando en dispositivos Android con el flujo de sincronización de n8n.
